## C. openclaw trading-engine 개선 우선순위

### P0 (즉시)

| 항목 | 현재 | 개선안 | 근거 |
|------|------|--------|------|
| TP/SL | TP +1%, SL -0.5% | **TP 3-5%, SL 2-3%** + ATR 동적 조정 | BTC 일일 변동 5-10%, 현재 너무 좁음 |
| 트레일링 스톱 | 없음 | **ATR 기반 트레일링 스톱** | 이익 확보 핵심 |
| 연속 학습 | 매번 처음부터 학습 | **체크포인트 이어서 학습** | FreqAI 방식, 효율 대폭 향상 |
| SAC 추가 | PPO+A2C+DDPG | **PPO+A2C+SAC** (DDPG→SAC 교체) | SAC가 크립토에서 일관적으로 최적 |

### P1 (단기)

| 항목 | 현재 | 개선안 | 근거 |
|------|------|--------|------|
| 온체인 피처 | yfinance만 | **Glassnode/CryptoQuant API 연동** | 예측 정확도 최대 향상 요인 |
| 감성분석 | 키워드 사전 | **FinGPT LoRA 모델** 또는 DPO 파인튜닝 | F1 87.62% vs 키워드 ~60% |
| 네트워크 | LSTM (SB3 기본) | **xLSTM** Actor/Critic | 시계열 처리 대폭 개선 |
| 레짐 감지 | 없음 | **HMM/변동성 클러스터링** → 전략 전환 | 강세/약세/횡보 동적 대응 |
| 정보 기반 바 | 시간봉 (일봉) | **CUSUM / Volume Bar** | 시간봉 대비 노이즈 감소 |
